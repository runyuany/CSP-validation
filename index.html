CSP Validation Test Suite

This site tests how browsers handle Content Security Policy directives with external CSS resources. Each scenario loads the same HTML but with different CSP headers.

Instructions:

  1. Click any scenario below
  2. Observe whether the test box turns green (CSS loaded) or remains unstyled (CSS blocked)
  3. Open browser console (F12) to view any CSP violation messages
  • Scenario 0: No CSP Header
  • Scenario 1: Bing's script-src (style-src absent)
  • Scenario 2: default-src 'self', style-src unspecified
  • Scenario 3: style-src 'self'
  • Scenario 4: style-src allowing specific external CSS domain
  • Scenario 5: style-src allowing specific CSS hostname
  • Scenario 6: style-src 'unsafe-inline' with no external allowances
  • Scenario 7: Broad style-src https:
  • Scenario 8: Combined default-src 'self' with style-src exception
